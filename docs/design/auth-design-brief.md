# ログイン / 新規登録画面 デザイン依頼書

## 1. サービス概要

**Anycast** は、AIを活用したポッドキャスト作成・配信プラットフォームです。ユーザーは台本を入力するだけで、AIがナレーションや対話形式の音声コンテンツを生成し、チャンネルを通じて配信できます。

**主な機能:**

- チャンネル作成・管理
- AIボイスによるエピソード生成
- キャラクター・BGM管理
- プレイリスト・ライブラリ機能（視聴履歴、いいね、フォロー）
- Studio（クリエイター向けダッシュボード）

**ターゲットユーザー:** ポッドキャストに興味があるクリエイターおよびリスナー（日本語話者）

**全体の雰囲気:** モダン、ミニマル、ダークテーマ基調のオーディオ関連サービス

## 3. 既存UIコンポーネント仕様

デザインはこれらの既存コンポーネント仕様に準拠してください。

### 3.1 ボタン

- **サイズ:** `sm` (30px) / `md` (36px) / `lg` (42px)
- **カラー:** `primary` / `secondary` / `danger`
- **種類:** `solid`（塗り） / `outline`（枠線） / `text`（テキストのみ）
- `primary + solid`: 背景 `#2e86c1`、テキスト白
- `secondary + solid`: 背景白、テキスト黒
- アイコン付きボタン対応（左右）
- ホバー時: 透明度80%、アクティブ時: 透明度70%

### 3.2 テキスト入力

- サイズ: `sm` / `md` / `lg`
- アイコン表示対応（左右）
- エラー状態（ボーダーが赤に変化）
- フォーカス時: リングスタイル表示

### 3.3 フォームラベル

- 「必須」マーク表示（赤色）
- 補助テキスト表示

## 4. デザイン対象画面

### 4.1 ログイン画面（`/login`）

**認証方法（優先度順）:**

> **重要: Google ログインを最も目立つ位置に大きく配置し、ユーザーが最初に目にする主要な認証手段として強調してください。** メールアドレス/パスワード認証は補助的な手段として、Google ログインの下部に配置する想定です。「または」などのセパレーターで区切り、視覚的な優先順位を明確にしてください。

1. **Google ログイン（優先・推奨）**
   - Google ブランドガイドラインに準拠したボタンデザイン
   - 画面上部または中央の目立つ位置に配置
   - ボタンサイズは他の要素より大きく

2. **メールアドレス + パスワード認証（補助）**
   - メールアドレス入力フィールド
   - パスワード入力フィールド
   - 「ログイン」ボタン

**その他の要素:**

- ページタイトル: 「ログイン」
- 新規登録ページへのリンク（「アカウントをお持ちでない方は 新規登録」）

### 4.2 新規登録画面（`/signup`）

**認証方法（優先度順）:**

> **重要: ログイン画面と同様に、Google アカウントでの登録を最優先に強調してください。**

1. **Google アカウントで登録（優先・推奨）**
   - 「Google で登録」ボタン
   - ログイン画面と同様に目立つ位置に

2. **メールアドレスでの新規登録（補助）**
   - 表示名
   - メールアドレス
   - パスワード
   - パスワード確認
   - 「新規登録」ボタン

**その他の要素:**

- ページタイトル: 「新規登録」
- ログインページへのリンク（「すでにアカウントをお持ちの方は ログイン」）

## 5. レイアウト要件

- 認証画面は独立したレイアウト（ヘッダー・サイドバーなし）
- 画面中央寄せのカード型レイアウトを想定
- サービスロゴを画面上部に配置
- モバイル（~640px）、タブレット（~1024px）、デスクトップ（1024px~）のレスポンシブ対応
